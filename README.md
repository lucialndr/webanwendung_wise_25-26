## Hinweis zur KI-gestützten Codegenerierung: <br>
Für die grundlegende Gestaltung haben wir zunächst Figma genutzt, um eine klare visuelle Struktur zu entwickeln. Bei der Umsetzung spezifischer Design Elemente, wie etwa komplexer Gradients, animierter Komponenten oder responsiver Layouts, erwies sich Claude Sonnet 4.5 als eine hilfreiche Unterstützung. Die KI konnte schnell CSS Variationen generieren und half dabei moderne Design Trends zu implementieren.
Die Kernarchitektur unserer Anwendung bestehend aus HTML Strukturen, JavaScript Logik und der Supabase Integration, haben wir jedoch eigenständig entwickelt. Dies war eine bewusste Entscheidung, da wir das grundlegende Verständnis der technischen Zusammenhänge für uns selber auch verstehen wollten . Wir haben die Datenbankstruktur, Row Level Security Policies und API-Endpunkte selber implementiert.
Die KI Assistenz kam eher bei konzeptionellen Fragen zum Einsatz: Wie strukturiert man am besten eine mehrstufige Formularvalidierung? Welche Best Practices gibt es für File Uploads in Supabase? Hier diente die KI als Recherchetool, ähnlich der klassischen Dokumentations- oder Stack Overflowsuche, jedoch mit dem Vorteil projektspezifischer Antworten.
Beurteilung der Praxistauglichkeit: Die von uns eingesetzte KI gestützte Codegenerierung ist für bestimmte Anwendungsfälle sehr praxistauglich, insbesondere bei repetitiven Design Aufgaben, der Generierung von Boilerplate Code oder beim Debuggen komplexer Fehler. Claude Sonnet 4.5 half dabei, Entwicklungszeit bei UI-Komponenten deutlich zu reduzieren. Allerdings zeigen sich auch klare Grenzen: Bei komplexen, projektspezifischen Anforderungen wie der Integration von Supabase Auth mit Custom Policies oder der Implementierung mehrstufiger Formulare mit File Handling benötigt man weiterhin ein tiefes Verständnis. Die KI kann Vorschläge machen, aber die finale Architekturentscheidung und Fehlersuche erfordern menschliches Urteilsvermögen.

# Nuxt Minimal Starter

Look at the [Nuxt documentation](https://nuxt.com/docs/getting-started/introduction) to learn more.

## Setup

Make sure to install dependencies:

```bash
# npm
npm install

# pnpm
pnpm install

# yarn
yarn install

# bun
bun install
```

## Development Server

Start the development server on `http://localhost:3000`:

```bash
# npm
npm run dev

# pnpm
pnpm dev

# yarn
yarn dev

# bun
bun run dev
```

## Production

Build the application for production:

```bash
# npm
npm run build

# pnpm
pnpm build

# yarn
yarn build

# bun
bun run build
```

Locally preview production build:

```bash
# npm
npm run preview

# pnpm
pnpm preview

# yarn
yarn preview

# bun
bun run preview
```

Check out the [deployment documentation](https://nuxt.com/docs/getting-started/deployment) for more information.
